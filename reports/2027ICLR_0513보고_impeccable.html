Optimizer-Induced Geometry Regimes for Feature-Based Uncertainty

현재 프로젝트 위치 NeurIPS 2026 결과를 ICLR 2027 optimizer-geometry 연구로 확장
현재 근거 문헌/기존 논문 근거 + 2 epoch smoke pipeline validation
주요 실험 축 optimizer choice, weight-decay coupling, NC/feature geometry, OOD detector
앞으로 할 일 200 epoch matched-protocol sweep와 multi-seed 결과
1

연구 배경과 현재 질문

기존 NeurIPS 2026 작업은 optimizer가 output-level reliability와 feature-based uncertainty를 서로 다르게 움직일 수 있음을 보였다. ICLR 2027 확장은 이 관찰을 SAM-family에서 optimizer family와 weight-decay coupling 축으로 넓히는 방향이다.

층위 현재 의미 근거 상태 보고서에서의 취급
NeurIPS 2026 출발점 SAM은 accuracy/calibration 또는 MSP/Energy를 유지·개선하면서 Mahalanobis/DDU 같은 feature-based detector를 악화시킬 수 있다. 확정 기존 논문 결과로 두고, ICLR의 직접 실험 결과와 섞지 않는다.
선행연구 연결점 optimizer choice와 coupled/decoupled weight decay가 Neural Collapse 및 representation geometry에 영향을 준다. 확정 optimizer -> geometry 축의 근거로 사용한다.
ICLR 2027 질문 optimizer-induced geometry regime이 Mahalanobis, DDU/GMM, kNN 같은 feature-based OOD detector의 성공과 실패를 어떻게 바꾸는가? 가설 full-scale 실험으로 검증할 중심 질문으로 둔다.
2

논리 구조

이 프로젝트의 논리는 optimizer를 성능 비교 항목으로만 보지 않고, 마지막 표현층 기하구조를 만드는 개입으로 본다는 점에 있다.

STEP 1 Optimizer choice / WD coupling SGD, Adam, AdamW, SAM-family, 그리고 custom coupled-decoupled interpolation.
STEP 2 Feature geometry / NC regime within-class compactness, class separation, NC0-NC4, covariance spectrum, feature norm.
STEP 3 Detector behavior Mahalanobis, Mahalanobis-L2, kNN, GMM/DDU, NC-inspired and hybrid detector families.
STEP 4 Paper-level claim Neural Collapse is not a scalar reliability certificate. Detector-relevant geometry must be decomposed.
논리의 핵심 분리

선행연구는 optimizer -> Neural Collapse/geometry를 뒷받침한다. 이 프로젝트가 새로 검증해야 하는 부분은 geometry -> downstream uncertainty detector behavior이다.

3

현재까지 확보된 근거

현재 근거는 네 층으로 나뉜다. 기존 논문 결과, 외부 문헌 근거, 로컬/서버 pipeline 검증, 그리고 아직 ICLR 실험으로 검증해야 할 가설이다.

항목 현재 상태 근거 아직 부족한 점
SAM에서 output-level reliability와 feature-based OOD가 분리될 수 있음 확정 소스/neurips2026_paper_context.md ICLR에서는 SAM-family를 넘어 optimizer family 전체로 확장해야 한다.
optimizer choice와 WD coupling이 NC/geometry에 영향을 줌 확정 소스/SRC-arxiv-2602.16642-v3.md, 소스/INDEX.md 이 문헌은 Mahalanobis, DDU, kNN 결과를 직접 검증하지 않는다.
M1A/M1B smoke pipeline이 train -> cache -> eval -> manifest까지 동작함 확정 reports/M1_SMOKE_STATUS_2026-05-13.md, 9개 run manifest 2 epoch smoke validation이므로 논문 evidence가 아니다.
custom adam_coupled_decoupled axis의 endpoint semantics 확정 reports/OPTIMIZER_ENDPOINT_SEMANTICS_2026-05-13.md endpoint full-training trajectory equality는 주장하지 않는다.
feature-based detector와 geometry metric contract 확정 reports/METRIC_DEFINITIONS.md 200 epoch runs에서 모든 output file이 contract대로 생성되어야 한다.
optimizer-induced geometry가 downstream detector 결과를 설명한다는 ICLR 중심 주장 가설 NeurIPS prior + optimizer/NC source가 논리적 간극을 형성 matched-protocol, multi-seed, full training 결과가 필요하다.
로컬 checkout 기준 주의

현재 Git-tracked checkout의 results/raw에는 .gitkeep만 보인다. 이 보고서는 raw metric JSON을 직접 재계산한 것이 아니라, manifest와 이미 작성된 smoke status report를 근거로 정리한다.

오늘 진행한 작업

5월 13일 작업의 중심은 연구 논리와 실험 pipeline을 분리해 정리하고, 다음 full-scale sweep으로 넘어갈 수 있는 최소 운영 조건을 확인한 것이다.

작업 확인한 내용 전체 논리에서의 의미 다음 확인
M1A smoke pipeline 정리 CIFAR-10, standard_cifar_resnet18, SGD, seed 0, 2 epoch run이 train/cache/eval/check를 통과했다. 서버에서 최소 실험 파이프라인이 작동한다는 운영 근거다. 200 epoch matched-protocol config로 확장한다.
M1B optimizer endpoint validation 정리 Adam, AdamW, custom r=0, custom r=1 smoke runs가 local-curie와 host 101에서 manifest로 등록됐다. AdamW-to-Adam interpolation axis를 실험 pipeline에 연결할 수 있다. r in {0.0, 0.25, 0.5, 0.75, 1.0} sweep 설계를 고정한다.
metric naming contract 확인 nc0_width_norm, inter_dist_l2, nc3_self_duality 등 revised names를 사용해야 한다. legacy metric name 혼동을 줄이고, 보고서와 evaluator output을 맞출 수 있다. full run evaluator output에서 contract를 자동 점검한다.
architecture boundary 확인 main evidence는 standard_cifar_resnet18와 standard_wrn_28_10_dropout03, no SN/no mod를 기준으로 둔다. DDU-specific architecture confound를 main claim에서 분리한다. DDU-style SN/mod는 appendix diagnostic로만 설계한다.
보고 산출물 품질 기준 분리 PRODUCT.md와 DESIGN.md가 HTML report 품질과 일관성에만 적용되도록 루트에 추가됐다. 보고서 포맷은 정교해지되 연구 해석 기준은 source/metric/manifest 문서가 유지한다. 이 HTML 보고서를 첫 적용 사례로 검증한다.

현재 해석과 남은 불확실성

지금 단계에서 말할 수 있는 것은 논문 결론이 아니라 연구 설계의 타당성과 pipeline 준비 상태다. 해석은 다음처럼 제한해 두는 것이 안전하다.

현재 해석

  • 해석 ICLR story는 NeurIPS prior와 optimizer/NC source 사이의 빈 공간에 놓인다.
  • 해석 AdamW-to-Adam interpolation은 accuracy만이 아니라 geometry와 detector behavior를 함께 볼 수 있는 가장 선명한 축이다.
  • 해석 DDU만으로 feature-based uncertainty를 대표하지 말고 Mahalanobis, kNN, tied/diag/shrinkage GMM, L2-normalized controls를 같이 봐야 한다.

남은 불확실성

  • 누락 200 epoch, multi-seed, matched-protocol 결과가 아직 없다.
  • 위험 smoke accuracy 차이는 pipeline sanity check일 뿐 optimizer 우열 근거가 아니다.
  • 위험 custom endpoint는 one-step semantics가 확인됐지만 PyTorch Adam/AdamW full trajectory clone으로 쓰면 안 된다.

7

용어

용어 짧은 설명 이 연구에서의 역할
Neural Collapse 학습 후반에 class feature와 classifier geometry가 특정 단순 구조로 수렴하는 현상. optimizer가 만드는 representation geometry를 분해해서 보는 핵심 언어다.
NC1 within-class variability collapse. 낮을수록 class 내부 feature가 더 조밀하다는 뜻으로 읽는다. Mahalanobis/DDU/kNN이 선호할 수 있는 compactness 축이다.
Coupled vs decoupled weight decay weight decay가 Adam moment 계산 안에 들어가는지, AdamW처럼 update와 분리되는지의 차이. AdamW-to-Adam interpolation의 직접 조작 변수다.
Mahalanobis class mean과 covariance precision을 이용해 feature가 ID class에 가까운지 보는 detector. feature covariance와 class compactness 변화에 민감하다.
DDU/GMM class-wise Gaussian density로 feature-space likelihood를 계산하는 detector family. covariance estimation 방식이 결과에 섞일 수 있어 tied/diag/shrinkage controls가 필요하다.
kNN ID train feature set에서 가까운 이웃까지의 거리를 이용하는 non-parametric detector. Gaussian assumption 없이 feature geometry 영향을 확인하는 비교축이다.
