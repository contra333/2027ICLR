# tasks/AGENTS.md

## Role

`tasks/` stores one-off task tickets when a request should be durable across Codex sessions.

## Rules

- Use `tasks/TEMPLATE.md` for new task notes.
- Keep each task focused on one goal.
- State success criteria and expected verification.
- If a task changes experiment settings, link the relevant config.
- If a task imports results, link the relevant result manifest.

## Completion

When a task is completed, record:

- files changed,
- commands/checks run,
- remaining risks,
- follow-up tasks if needed.
